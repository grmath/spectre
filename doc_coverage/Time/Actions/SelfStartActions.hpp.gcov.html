      36             : /// Definition of the integrator self-starting procedure.
      37             : ///
      38             : /// The self-start procedure generates \f$N\f$ function values of
      39             : /// accuracy \f$O\left((\Delta t)^N\right)\f$, where \f$N\f$ is the
      40             : /// negative of the initial slab number.  To generate values, it
      41             : /// requires the time stepper to be a multistep integrator that will
      42             : /// produce an order-\f$k\f$-accurate result given \f$k-1\f$ history
      43             : /// values.
      44             : ///
      45             : /// If the integrator is started from analytic history data or
      46             : /// requires no history (such as for a substep integrator), then the
      47             : /// initial slab number can be set to zero and no self-start steps
      48             : /// will be taken.
      49             : ///
      50             : /// \details
      51             : /// To self-start a multistep integrator, the function is integrated
      52             : /// repeatedly with increasing accuracy.  A first order integrator
      53             : /// (Euler's method) requires no history values, so it can be used,
      54             : /// starting from the initial point, to generate a
      55             : /// first-order-accurate value at a later time.  We then reset to the
      56             : /// start conditions and use the new "history" value (at a discarded
      57             : /// point later in time) to take two steps with a second-order method.
      58             : /// These values are second-order-accurate despite the history only
      59             : /// being first-order because the calculation of the change in the
      60             : /// value multiplies the previous derivatives by a factor of
      61             : /// \f$\Delta t\f$.  The time and value are then again reset to their
      62             : /// starting values and we start again at third order, and so on.
      63             : ///
      64             : /// The choice of performing the low-order integrations in the same
      65             : /// direction as the main integration makes this a _forward_
      66             : /// self-start procedure, as opposed to a _backward_ procedure that
      67             : /// produces values for times before the start time.  The primary
      68             : /// advantage of the forward version is that the solution is
      69             : /// guaranteed to exist after the start time, but not before.  It also
      70             : /// makes bookkeeping easier, as the reset after each order increase
      71             : /// is to the initial state, rather than to a time one step further
      72             : /// back for each order.  It does have the disadvantage, however, of
      73             : /// leaving a non-monotonic history at the end of the procedure, which
      74             : /// the main evolution loop must be able to handle.
      75             : ///
      76             : /// Each time the state is reset the slab number is increased by one.
      77             : /// This ensures that the evaluations are considered to be ordered in
      78             : /// their evaluation order, even though they are not monotonic in
      79             : /// time.  When the slab number reaches zero the initialization
      80             : /// procedure is complete and history appropriate for use for an
      81             : /// integrator of order \f$N+1\f$ has been generated.
      82             : ///
      83             : /// The self-start procedure performs all its evaluations before the
      84             : /// end of the first time step of the main evolution.  It is important
      85             : /// that none of the early steps fall at the same time as the
      86             : /// self-start history values, so the main evolution should not
      87             : /// decrease its step size on the first step after the procedure.
      88             : /// Additionally, the history times will not be monotonicly increasing
      89             : /// until \f$N\f$ steps have been taken.  The local-time-stepping
      90             : /// calculations require monotonic time, so local time-stepping should
      91             : /// not be initiated until the self-start values have expired from the
      92             : /// history.  These restrictions on step-size changing are checked in
      93             : /// the TimeStepper::can_change_step_size method.
